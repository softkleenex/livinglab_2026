# 포트폴리오 활용 가이드 (MDGA 프로젝트)

본 문서는 이력서, 경력 기술서 및 포트폴리오 작성을 위해 MDGA(Universal Data Engine) 프로젝트의 핵심 성과와 기술적 문제 해결 과정을 정리한 가이드입니다. 본인의 지원 직무(Backend, Frontend, Full-Stack, AI Engineer)에 맞게 내용을 발췌하여 사용하세요.

## 1. 프로젝트 한 줄 소개
**"파편화된 농기계 데이터와 작물 생육 데이터를 결합하여 고부가가치 합성 데이터를 생성하는 스마트팜 특화 AI 데이터 파이프라인 구축"**

## 3. 핵심 기능 및 성과 (비즈니스 임팩트)

### 🌾 농림/스마트팜 특화 데이터 파이프라인
* **데이터 사각지대 해소:** 수기로 작성되던 영농일지나 버려지는 농기계 데이터를 멀티모달 AI(Gemini Vision)로 분석하여 자동 디지털 자산화.
* **생육-농기계 데이터 결합:** 분절되어 있던 작물 생육 데이터와 농기계 가동 데이터를 결합하는 파이프라인 설계.

### 🌐 지역 특화 기후 대응 시뮬레이터 (Twin Map & Dashboard)
* **기후 변화(아열대화) 대응:** 대구 사과 등 지역 특산물 데이터를 기반으로 기상청 및 농진청 데이터를 연동하여 미래 기후 변화에 따른 재배 적지 및 수확량 시뮬레이션 제공.
* **지역 계층 롤업 엔진:** 개별 농가의 데이터를 동 -> 구 -> 시 단위로 자동 집계하여 지역 경제 및 농업 건강도를 지도상(Twin Map)에 시각화.

### 🧠 고부가가치 합성 데이터(Synthetic Data) 생성 및 B2B 거래소
* **합성 데이터 생성:** 부족한 실측 데이터를 보완하기 위해 기존 데이터와 공공 기상/토양 데이터를 결합하여 고품질의 합성 데이터를 생성.
* **데이터 셀러 BM 확립:** 구축된 합성 데이터를 B2B 시장(AI 허브, 기업 연구소 등)에 판매할 수 있는 합성 데이터 거래소 모듈(B2B Market) 구현.

## 4. 기술적 문제 해결 과정 (Troubleshooting & Architecture)
*이 부분은 면접 및 포트폴리오에서 가장 강력한 어필 포인트가 됩니다.*

### 🚀 1. 시스템 아키텍처 한계 극복 (모놀리식 -> 3NF 정규화)
* **문제:** 초기 MVP는 메모리 상에서 트리 구조를 관리하는 상태 유지(Stateful) 서버 형태로, 트래픽 증가 시 Scale-out이 불가능했으며 데이터 무결성이 보장되지 않음.
* **해결:** Postgres 기반의 관계형 데이터베이스로 전면 전환. Region, Store, DataEntry 등 테이블을 3정규화(3NF)하고 FK와 `ON DELETE CASCADE`를 적용. 결과적으로 상태를 DB로 완전히 위임(Stateless)하여 무한 확장이 가능한 백엔드 아키텍처 완성.

### 🤖 2. LLM의 'Safety Alignment' 우회 및 시스템 제어 체계 구축
* **문제:** 사용자가 챗봇에게 "방금 올린 데이터 삭제해줘"라고 요청하면 LLM이 "권한이 없다"며 안전망(Safety Alignment)에 의해 거부하는 현상 발생.
* **해결:** AI를 '대화용 페르소나'와 '시스템 제어용 의도 분석기(Intent Parser)' 두 개의 레이어로 분리. 의도 분석기는 감정 없는 JSON 스키마만을 출력하도록 강제하고, 백엔드는 이 JSON(`action: DELETE`)을 해석하여 실제 DB 트랜잭션을 실행함. 안전한 AI 기반 시스템 CRUD 제어를 구현.

### 🔒 3. Google Drive 데이터 레이크 연동 시 권한 에러 해결 (OAuth 최소 권한)
* **문제:** 백엔드에서 Drive 데이터를 삭제할 때 루트 폴더 권한이 있음에도 `403 Forbidden` 발생.
* **해결:** 전역 접근 권한을 요구하는 대신, `auth/drive.file` 스코프의 한계를 파악함. DB에 저장된 고유 `hash_val`과 Drive File ID를 매핑하여 앱이 생성한 데이터만 외과 수술처럼 정밀하게 제어하도록 격리된 권한 아키텍처 설계.

### ⏱ 4. 소규모 팀의 엔터프라이즈급 구현 (AI + CI/CD 자동화)
* **문제:** 제한된 인력과 시간 내에 멀티모달 분석부터 DB 트랜잭션, 대시보드 시각화까지 방대한 작업을 처리해야 함.
* **해결:** GitHub Actions를 활용한 CI/CD 파이프라인 자동화와 Cloudflare Pages, Render 등의 PaaS를 적극 활용하여 배포 프로세스를 간소화하고, AI 코파일럿(Gemini 2.5 Pro) API를 결합하여 소규모 팀으로도 도입 비용을 70% 절감하는 아키텍처 구축.

## 5. 면접 예상 질문 & 답변 가이드
- **Q: 농기계 데이터와 생육 데이터를 구체적으로 어떻게 결합했나요?**
  - A: "농기계의 가동 시간 및 센서 로그를 수집하고, 이에 대응하는 시기의 작물 생육 지표(사진 스캔, 영농일지)를 매핑하여 상관관계를 분석할 수 있는 통합 JSON 포맷으로 파이프라인을 구축했습니다."
- **Q: 합성 데이터(Synthetic Data)를 어떻게 생성하고 활용했나요?**
  - A: "수집된 샘플 데이터가 부족한 한계를 극복하기 위해, Hugging Face의 농업 데이터와 기상청의 시나리오 데이터를 프롬프트에 주입하여 다양한 기후 조건 하에서의 예상 수확량 및 생육 상태를 담은 합성 데이터를 AI가 자동 생성하도록 했습니다."
- **Q: LLM을 단순히 챗봇으로 쓰지 않고 시스템에 어떻게 녹여냈나요?**
  - A: "비정형 데이터의 정형화(AI-Ready) 파이프라인과 더불어, 사용자의 자연어 명령을 시스템 제어용 JSON으로 변환하는 '2단계 파싱 봇' 구조를 도입하여 사용자 명령으로 DB 레코드가 직접 수정/삭제되도록 구현했습니다."
