# MDGA: Smart Farm Pivot Strategy

## 1. 개요 (Overview)
본 문서는 2026.04.08 진행된 '지니스팜(스마트팜 운영자)' 인터뷰를 기반으로, MDGA(Multi-dimensional Data Gathering Architecture) 앱의 핵심 가치와 발전 방향성을 '일반 상권'에서 **'B2B 스마트팜 / 농업 생산자 맞춤형'**으로 확장 및 피벗(Pivot)한 전략을 문서화합니다.

## 2. 인터뷰 인사이트 요약
*   **현재 상태:** 수기 영농일지(생산)와 토글 플랫폼(출고/주문)으로 데이터가 분산되어 관리됨.
*   **핵심 페인 포인트 (Pain Point):** 매출액과 실제 택배 출고 수량을 한 번에 통합해서 볼 수 있는 직관적인 대시보드의 부재.
*   **가장 큰 니즈 (Killer Feature):** 단순 데이터 조회가 아닌, **연평균 판매 데이터를 분석하여 다음 달 재배량(파종량)을 추천해 주는 기능**.
*   **사업적 기회:** 해당 기능(AI 추천)이 구현될 경우 실질적인 사용 의사와 데이터 제공 의사가 매우 높음. (지불 의사 확인)

## 3. 앱 반영 및 고도화 사항 (v1.5 업데이트)

인터뷰 결과를 즉각적으로 프론트엔드 앱 로직에 반영하여 아래 기능을 구현 완료했습니다.

### 3.1. 산업군(Industry) 기반 Context Switching
*   초기 온보딩(객체 선택) 시, 사용자가 산업군을 `스마트팜` 또는 `농업`으로 입력할 경우, 메인 대시보드(Personal Tab)의 UI가 완전히 **생산자 맞춤형 대시보드**로 전환되도록 로직을 분기(`isSmartFarm` 플래그)했습니다.

### 3.2. Data Hub (데이터 통합 허브) UI
*   분산된 데이터 입력을 일원화하기 위해 3가지 주요 연동 채널 버튼을 신설했습니다.
    1.  **수기 영농일지 연동:** 사진/텍스트 기반 비정형 데이터 수집
    2.  **쇼핑몰 (토글) 연동:** 일별 주요 주문/매출 데이터 수집
    3.  **택배사 API 연동:** 실 출고량 크롤링

### 3.3. 듀얼 지표 대시보드 (매출 vs 출고)
*   일간 매출 금액 현황과 함께, AI가 예측한 **'익월 추천 파종량'**을 메인 지표로 나란히 배치했습니다. 생산자가 가장 알고 싶어 하는 결론을 화면 상단에 즉각 노출합니다.

### 3.4. AI 파종량 추천 분석 리포트 (Report Modal)
*   단순히 '추천량' 숫자만 보여주는 것을 넘어, [주간 리포트 발행] 버튼 클릭 시 **스마트팜 전용 AI 분석 리포트 팝업**이 출력되도록 `ReportModal`을 대폭 수정했습니다.
*   **리포트 구성 요소:**
    *   현재 생산/출고 동향 요약 및 데이터 신뢰도(Trust Index) 평가
    *   **익월 파종/재배 추천량 및 예측 근거 (기상청 데이터, 온라인 키워드 트렌드 결합)**
    *   즉시 실행 가능한 액션 플랜 (Action Plan) 제공

## 4. 향후 발전 방향
*   **백엔드 API 고도화:** 현재 Mock 데이터로 구성된 프론트엔드의 스마트팜 리포트 출력을 실제 백엔드 LLM 프롬프트 템플릿(예: `스마트팜 특화 프롬프트`)과 연결하는 작업이 필요합니다.
*   **데이터 파이프라인 연동:** 실제 토글(쇼핑몰) API 및 로젠/CJ 등 주요 택배사 배송 조회 API와의 연동 테스트 코드를 작성해야 합니다.
