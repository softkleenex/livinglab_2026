# 15. MDGA System Encapsulation & Final Architecture Report

본 문서는 MDGA(Universal Data Engine) 프로젝트의 최종 배포 아키텍처, 데이터베이스 정규화(Normalization) 과정, 그리고 프론트엔드 UX/UI 개편 및 트러블슈팅 사례를 총망라한 기술 명세서입니다.

---

## 1. 아키텍처 개편 및 데이터베이스 정규화 (Refactoring)

초기 MVP 단계에서는 빠른 시연을 위해 단일 테이블(`DataEntry`)에 모든 로그를 쌓고, 서버의 메인 메모리(RAM)에 전체 지역구 트리를 띄우는 **Monolithic & Stateful** 방식을 채택했습니다. 하지만 상용 수준의 트래픽을 감당하기 위해 백엔드를 전면 재설계했습니다.

### 1.1. Stateless Hierarchy Engine
서버 메모리에 의존하던 트리 렌더링 로직을 폐기하고, API가 호출될 때마다 실시간으로 관계형 데이터베이스(RDBMS)를 조회하여 트리를 조립하도록 엔진을 **Stateless 구조**로 개편했습니다. 
- **효과:** 클라우드 환경(Render 등)에서 백엔드 서버를 여러 대(Scale-out) 늘려도 데이터가 불일치하는 현상이 원천 차단되었습니다.

### 1.2. 3단 정규화 (Region - Store - DataEntry)
- **`Region` 테이블:** 시, 구, 동, 도로명을 `parent_id` 외래키(Self-referencing)로 묶어 무한한 깊이의 지역 트리를 표현합니다.
- **`Store` 테이블:** 사용자의 사업장을 객체화하여, 누적 가치(Total Value)와 신뢰 지수(Trust Index)를 관리합니다.
- **`DataEntry` 테이블:** 실제 업로드된 텍스트와 Google Drive 링크, AI 인사이트를 담습니다.

**🔥 트랜잭션 무결성과 영구 삭제 로직 (`ON DELETE CASCADE`)**
기존에는 객체 자체를 지울 수 없었으나, 3단 정규화를 통해 **사업장 통째로 삭제 기능(`DELETE /api/ingest/store`)**을 구현했습니다.
사업장을 삭제하면 하위의 수십 개 DataEntry가 한꺼번에 지워지고, 부모 Region이 가졌던 '자산 총합(Total Value)'에서 해당 사업장의 가치만큼을 오차 없이 차감(Rollback)하는 완벽한 무결성을 달성했습니다.

## 2. Google Drive Data Lake 연동 최적화

MDGA 시스템은 유저가 올린 데이터를 단순히 DB 텍스트로만 남기지 않고, **Google Drive를 거대한 데이터 레이크(Data Lake)**로 활용합니다.

- **동적 폴더 라우팅:** 업로드 시 사용자의 위치(`대구광역시/북구/산격동/지니스팜`)를 인식하여 구글 드라이브 내에 동일한 계층 구조의 폴더를 실시간으로 자동 생성합니다.
- **권한 격리 (appNotAuthorizedToChild 해결):** 
  초기에는 시스템 리셋 시 드라이브 폴더 전체를 날리려다 `403 Forbidden` 에러에 직면했습니다. 원인은 OAuth Scope가 `drive.file`(앱이 만든 파일만 조작 가능)로 제한되어 있었기 때문입니다. 
  이를 해결하기 위해 폴더 전체를 날리지 않고, DB에 기록된 해시값(`hash_val`)을 바탕으로 **정확히 MDGA가 생성한 파일만 조회(Search)하여 타겟팅 삭제**하는 안전한 방식으로 구조를 변경했습니다.

## 3. Two-Step AI Parsing (챗봇 명령 제어)

MDGA Copilot 챗봇은 일반적인 대화를 넘어 **시스템을 직접 조작(데이터 삭제/수정/생성)**하는 권한을 가집니다.

- **문제 (AI의 거절):** LLM(Gemini)에게 "데이터를 지워줘"라고 하면, AI 윤리 및 역할 필터가 작동하여 *"저는 비서라 시스템에 접근할 권한이 없습니다"* 라며 지속적으로 명령을 거부했습니다.
- **해결 (파서와 페르소나의 분리):**
  1. **의도 분석기 (JSON Parser):** 페르소나를 모두 제거하고, `ChatActionResponse` (TypedDict) 스키마를 강제 주입하여 사용자의 의도를 `DELETE`, `CREATE`, `MODIFY` 등 Action 객체로만 파싱하도록 지시했습니다.
  2. **시스템 트랜잭션:** 파싱된 JSON을 파이썬 백엔드가 넘겨받아 실제 DB와 Drive의 삭제를 수행합니다.
  3. **친절한 비서 (Copilot):** 일반적인 대화일 경우에만 컨설턴트 페르소나를 장착한 AI가 응답을 생성하도록 흐름을 쪼갰습니다.

## 4. 프론트엔드 UX/UI 개선 (Data Lifecycle)

- **명확한 원본 데이터 트래킹:** 기존 UI에서는 사용자가 피딩한 원본(Raw Data) 없이 AI 인사이트만 출력되었습니다. 이를 개선하여 타임라인 카드 안에 **[업로드한 원본]**과 **[AI 피드백]**, 그리고 **[첨부 파일 드라이브 링크]**가 한눈에 들어오도록 캡슐화된 UI로 개편했습니다.
- **AI 챗봇 파일 선택 기능:** 챗봇 화면에서 사용자가 올렸던 기록들을 체크박스(✅)로 선택하고, *"내가 선택한 이 파일을 삭제해/수정해"* 라고 말할 수 있는 컨텍스트 주입 UI를 신설했습니다.

## 5. 결론 및 성과
MDGA 플랫폼은 단순한 해커톤 목업을 넘어, 완벽한 RDBMS 설계, 분산 처리가 가능한 Stateless 아키텍처, 100% 무결성을 자랑하는 롤백 트랜잭션, 그리고 LLM을 활용한 안전한 자율 제어(Action Parsing) 로직이 탑재된 **엔터프라이즈 B2B SaaS**로 성공적으로 고도화되었습니다. 🚀
