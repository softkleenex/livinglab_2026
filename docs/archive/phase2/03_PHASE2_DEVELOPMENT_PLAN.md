# Phase 2 Development & Engineering Plan

## 1. 개요 (Objective)
본 문서는 Phase 1의 '범용 데이터 허브' 아키텍처 한계를 벗어나, Phase 2의 핵심 목표인 **[Agricultural AX (합성 데이터) + 지역 소상공인 B2B 매칭 플랫폼]** 구현을 위한 종합 개발 계획서입니다. 기존 틀에 얽매이지 않고, 외부 공공 API 및 상용 서비스를 적극 도입하여 실효성 있는 실증(Living Lab) 모델을 단기간 내에 구축합니다.

## 2. 외부 API 및 데이터 연동 전략 (Data Sourcing)
Phase 2의 핵심은 자체 데이터베이스 구축을 넘어, 방대한 공공 데이터를 실시간으로 끌어와 가공(AI-Ready) 및 합성(Synthetic)하는 것입니다.

### 2.1. Agricultural AX (합성 데이터 생성용)
*   **기상/기후 데이터:** 기상청 API (단기예보, ASOS, 방재기상관측, 기후변화 시나리오 RCP) 연동.
*   **농업/토양 데이터:** 농촌진흥청 API (전자토양도, 표준 시비량), 농림축산식품부 API (작물 생산량).
*   **유통/가격 데이터:** 한국농수산식품유통공사(aT) 실시간 도매시장 경매 가격 API (수급 불균형 지표용).
*   **가공 파이프라인:** 수집된 위 이종(Heterogeneous) 데이터들을 Backend(Python/FastAPI)에서 Pandas/Numpy로 전처리 후, Gemini 모델을 통해 결측치를 보간하고 예측 시나리오(JSON)를 뱉어내는 배치(Batch) 파이프라인 구축.

### 2.2. B2B 매칭 및 리버스 로지스틱스용
*   **지도/공간 정보 (Twin Map):** 기존 단순 지도에서 벗어나, 카카오맵 API 또는 네이버 지도 API를 연동하여 실제 대구/경북 지역의 동네 단위 상권 및 농가 위치 기반 매칭(클러스터링) 렌더링.
*   **알림/커뮤니케이션:** 알림톡(카카오) API 연동 등을 고려하여 소상공인 공동구매 달성, 또는 B급 농산물 등록 시 즉각적인 푸시 알림 체계 구상.

## 3. 아키텍처 및 시스템 개편 방향

### 3.1. 백엔드 (Backend - FastAPI & Supabase)
기존 3NF 구조(Region - Store - DataEntry)를 기반으로 도메인 특화 모델로 확장합니다.
*   **신규 테이블 정의 (Schema Update):**
    *   `Products` (B급 농산물, 농기계, 부산물 등 거래/매칭 대상 객체)
    *   `Matching` (공동구매, 대여, 수거 등 B2B 트랜잭션 기록)
    *   `SyntheticData` (백엔드 배치 작업으로 생성된 기상-수확량-가격 융합 데이터 캐싱)
*   **AI Agent 고도화:** 기존의 단순 요약을 넘어, 이미지(B급 농산물 상태)를 분석하여 상태 등급(A/B/C)과 적정 가공 용도(주스용, 잼용)를 JSON으로 반환하는 `Function Calling` 템플릿 세분화.

### 3.2. 프론트엔드 (Frontend - React & Vite)
단일 대시보드를 버리고 유저 페르소나에 따른 Multi-View를 제공합니다.
*   **생산자(농가) View:** 기상/수급 위기 경보 대시보드, 유휴 장비/B급 농산물 등록 폼.
*   **소상공인 View:** 동네 단위 식자재 공동구매 현황판, 카페 부산물 수거 요청 맵.
*   **행정/연구원 View:** 지역별 합성 데이터(AI-Ready Data) 다운로드, 수급 불균형/기상 위기 Twin Map 오버레이.

## 4. 단계별 개발 마일스톤 (Action Plan)

*   **Step 1: 외부 API 연동 및 데이터 수집 파이프라인 구축 (Week 1)**
    *   기상청, aT 등 공공데이터 포털 API 키 발급 및 백엔드 연동 모듈(Scheduler/Cron) 구현.
    *   수집된 데이터를 바탕으로 MVP 수준의 '수확량 변동 예측' 또는 '수급 위험 지표' 합성 로직(Gemini 결합) 테스트.
*   **Step 2: DB 스키마 마이그레이션 및 B2B 매칭 로직 개발 (Week 2)**
    *   Supabase에 `Products`, `Matching` 테이블 추가.
    *   B급 농산물 등록, 공동구매 수요 취합을 처리할 REST API 엔드포인트 개발.
*   **Step 3: 프론트엔드 페르소나 분리 및 Twin Map 고도화 (Week 3)**
    *   카카오/네이버 지도 기반의 동네 상권 매칭 UI 구현.
    *   농민 vs 소상공인 계정 분리에 따른 맞춤형 대시보드 UI 연동.
*   **Step 4: 통합 테스트 및 리빙랩 현장 검증 (Week 4)**
    *   실제 인터뷰이(농민/소상공인) 대상 파일럿 테스트 진행, 피드백 기반 버그 픽스 및 사용성 개선.

## 5. 완료 보고 (Completion Report)
- **멘토 피드백 반영 (Scope Narrowing & Open-Source SDV 적용):** 
  - "제공하겠다는 서비스가 너무 많다"는 피드백을 수용하여, 핵심 기능인 **A-1, A-2, A-3 (기후/수확량/수급 예측 합성데이터)** 및 **B-1, B-2 (B급 농산물 & 동네 공동구매)**에 집중하고, 나머지 기능(A-4, A-5, B-4, B-5)은 Phase 3(Future Work)로 우선순위를 하향 조정(Scope-out) 하였습니다.
  - "오픈소스 합성데이터 솔루션 사용" 권고에 따라, 오픈소스 라이브러리인 **SDV (Synthetic Data Vault)**를 백엔드 파이프라인에 도입하여 농업 기상/수확량 시뮬레이션 합성데이터를 성공적으로 생성(`scripts/generate_synthetic_agriculture_data.py`)하고 테스트를 완료하였습니다.
- **파트 A (Agricultural AX 핵심):** A-1 수확량 변동 예측, A-2 기후 대응 시뮬레이터, A-3 수급 불균형 위험 지표 (SDV 연동 및 백엔드 파이프라인 구현 완료)
- **파트 B (B2B 매칭 핵심):** B-1 B급 농산물 직거래, B-2 동네 단위 공동구매, B-3 원터치 변환기 (UI 및 API 연동 완료)
- **파트 C (추가 파이프라인):** C-1 통합 파이프라인, C-2 라벨링 자동화 엔진 (엔드포인트 연동 완료)
- **테스트 및 검증:** `pytest` 전체 통과 및 `dev.sh` 정상 구동 확인.
- **배포:** 프론트엔드(`deploy_frontend.sh`) 및 백엔드(`deploy_prod.sh`) 배포 환경 적용 완료.
